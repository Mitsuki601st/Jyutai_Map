# 1. クラス図

## 1.1 モデル / DbContext / コントローラ全体図

```mermaid
classDiagram

    class TrafficSurvey {
        +int Id
        +int CongestionLevel
        +string Comment
        +DateTime CreatedAt
    }

    class ApplicationDbContext {
        +DbSet~TrafficSurvey~ TrafficSurveys
        +OnModelCreating(ModelBuilder)
    }

    class TrafficvolumeController {
        -ApplicationDbContext _db

        +Index() IActionResult
    }

    class SurveyController {
        -ApplicationDbContext _db

        +Create() IActionResult
        +Create(TrafficSurvey) IActionResult

        +Edit(int id) IActionResult
        +Edit(int id, TrafficSurvey) IActionResult

        +Delete(int id) IActionResult
        +DeleteConfirmed(int id) IActionResult
    }

    TrafficvolumeController ..> TrafficSurvey
    SurveyController ..> TrafficSurvey

    TrafficvolumeController ..> ApplicationDbContext
    SurveyController ..> ApplicationDbContext

    ApplicationDbContext o-- TrafficSurvey
```

---

## 1.2 クラスの責務

| クラス                       | 種類        | 責務                 |
| ------------------------- | --------- | ------------------ |
| `TrafficSurvey`           | エンティティ    | 渋滞情報アンケートデータを保持する  |
| `ApplicationDbContext`    | DbContext | EF Core の DB 接続管理  |
| `TrafficvolumeController` | コントローラ    | 地図表示と渋滞度表示         |
| `SurveyController`        | コントローラ    | アンケートの投稿 / 編集 / 削除 |

# 2. 処理フロー

## 2.1 地図表示（F-01）

```mermaid
sequenceDiagram

    actor User as ユーザー
    participant Browser as ブラウザ
    participant Controller as TrafficvolumeController
    participant DB as PostgreSQL
    participant Maps as Google Maps API

    User->>Browser: トップ画面へアクセス

    Browser->>Controller: GET /

    Controller->>DB: 渋滞データ取得
    DB-->>Controller: 渋滞データ一覧

    Controller-->>Browser: View返却

    Browser->>Maps: Google Maps 初期化
    Browser->>Browser: 渋滞度を色分け表示
```

---

## 2.2 アンケート投稿（F-02）

```mermaid
sequenceDiagram

    actor User as ユーザー
    participant Browser as ブラウザ
    participant Controller as SurveyController
    participant DB as PostgreSQL

    User->>Browser: フォーム入力(渋滞度 / コメント)

    Browser->>Controller: POST /Survey/Create

    Controller->>Controller: ModelState検証

    alt 入力エラー
        Controller-->>Browser: エラー付きで再表示

    else 正常
        Controller->>DB: INSERT
        DB-->>Controller: 保存完了

        Controller-->>Browser: トップ画面へリダイレクト
    end
```

---

## 2.3 アンケート編集（F-03）

```mermaid
flowchart TD

    A[編集画面表示]

    B[DBから対象データ取得]

    C{データ存在?}

    C -- No --> D[404エラー]

    C -- Yes --> E[編集内容入力]

    E --> F[ModelState検証]

    F -- NG --> G[エラー表示]

    F -- OK --> H[DB更新]

    H --> I[トップ画面へ遷移]
```

---

## 2.4 アンケート削除（F-04）

```mermaid
flowchart TD

    A[削除ボタン押下]

    B[削除確認画面表示]

    C{削除実行?}

    C -- No --> D[トップ画面へ戻る]

    C -- Yes --> E[DBから削除]

    E --> F[トップ画面へ遷移]
```

# 3. テーブル定義書

## 3.1 ER図

```mermaid
erDiagram

    traffic_surveys {

        int id PK
        int congestion_level
        text comment
        timestamp created_at
    }
```

---

## 3.2 テーブル定義

### 3.2.1 `traffic_surveys`

| カラム              | 型         | NULL     | デフォルト    | 説明       |
| ---------------- | --------- | -------- | -------- | -------- |
| id               | integer   | NOT NULL | identity | 主キー      |
| congestion_level | integer   | NOT NULL | -        | 渋滞度（1〜5） |
| comment          | text      | NULL     | -        | コメント     |
| created_at       | timestamp | NOT NULL | now()    | 投稿日時     |

---

## 3.3 制約

| 種類      | 内容                               |
| ------- | -------------------------------- |
| 主キー     | id                               |
| CHECK制約 | congestion_level BETWEEN 1 AND 5 |

# 4. バリデーション設計

| 項目   | 条件      | エラーメッセージ              |
